# Task: Core Logic and Auth Refactoring

## Контекст
Необходимо внедрить логику алиасов моделей из `gemini-cli` и улучшить безопасность работы с токенами.

## Шаги реализации
1.  **Auth**:
    *   Обновить `auth/credentials.py`: использовать переменные окружения для Client ID/Secret.
    *   Добавить логирование процесса рефреша токенов.
    *   Добавить проверку "живучести" фонового потока.
2.  **Models**:
    *   Реализовать `resolve_model` в `core/models.py` с поддержкой `auto`, `pro`, `flash`.
    *   Добавить поддержку Gemini 3 (preview) через суффиксы или заголовки.
3.  **Error Handling**:
    *   Заменить `except: pass` в `api/openai/transform.py` и `api/gemini/routes.py` на явную обработку и логирование.

## Критерии готовности
- [ ] Алиасы `auto`, `pro`, `flash` работают корректно.
- [ ] Секреты не захардкожены в `auth/`.
- [ ] Ошибки трансформации логируются, а не игнорируются.
