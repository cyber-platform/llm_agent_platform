# Task: API Performance and Dynamic Models

## Контекст
Текущая реализация создает новый HTTP клиент на каждый запрос. Также список моделей статичен.

## Шаги реализации
1.  **HTTP Pool**:
    *   Создать глобальный `httpx.Client` в `main.py` или через провайдер.
    *   Обновить все роуты для использования общего клиента.
2.  **Dynamic Models**:
    *   Реализовать эндпоинт `GET /v1/models`.
    *   Для Vertex AI: запрашивать список через Google API.
    *   Для Quota: возвращать список на основе нового конфига моделей.
3.  **Final Cleanup**:
    *   Удалить неиспользуемые импорты.
    *   Проверить соответствие OpenAI API (поля `id`, `object`, `created`).

## Критерии готовности
- [ ] Используется пул соединений (проверяется по логам/коду).
- [ ] Эндпоинт `/v1/models` возвращает корректный JSON.
- [ ] Прокси успешно обрабатывает запросы к разным моделям через алиасы.
