Где в системе живут role-specific subsets для rules, skills и MCP tools, которые требуются термином agent-role?
Тут есть технический момент. agent system kilo и некоторые другие не поддерживают agent-role specific rules, skills, mcp tools.
Сейчас все сводится к идее чтобы у HyperGraph сервиса который всем этим управляет был hyper-graph-mcp.
При материализации агенту в системный промпт будет добавлена запись - так раньше работал старый kilo

перечень  specific role skills и его описание кроткое из frontmatter чтобы агент знал что такие навыки у него есть.

И дальше инструкция:
- чтобы прочитать нужный skill - запроси его в hyper-graph-mcp.get_skill(<skill_name>). В ответ придет полный путь к папке с этим skill
- чтобы использовать инструмент напиши какой инструмент и как ты хочешь вызвать в hyper-graph-mcp.use_tool(<tool_name>, строковый вызов tool с параметрами)
Разделение инструментов будет через hyper-graph-mcp который является еще и прокси для доступа к mcp tools
