# Task: Full Audit and Implementation of Transformation Logic (Gemini CLI Parity)

## Контекст
Необходимо обеспечить полную поддержку возможностей Gemini API на уровне, который предоставляет `gemini-cli`. Vertex AI поддержка на данном этапе является вторичной.

## Шаги реализации
1.  **Audit (CLI Parity)**:
    *   Изучить `gemini-cli/packages/core/src/code_assist/converter.ts` для понимания маппинга запросов.
    *   Проверить поддержку типов контента: текст, изображения, PDF (через `inlineData`).
    *   Уточнить работу с `system_instruction` и их приоритет.
    *   Сверить реализацию `thoughtSignature` и `thinkingConfig` с актуальным кодом CLI.
2.  **Implementation**:
    *   Добавить поддержку PDF и других медиафайлов, поддерживаемых CLI.
    *   Улучшить маппинг `finish_reason` (обработка `SAFETY`, `RECITATION` и т.д. в формате OpenAI).
    *   Реализовать поддержку Google Search Grounding (если CLI это использует через `tools`).
    *   Обеспечить корректную передачу `thinkingBudget` (лимит 8192 как в CLI).
3.  **Testing**:
    *   Написать тесты, имитирующие запросы от `gemini-cli` и проверяющие корректность трансформации в формат Google API.

## Критерии готовности
- [ ] Прокси поддерживает все возможности трансформации, доступные в `gemini-cli`.
- [ ] Корректно обрабатываются специфические для Gemini причины завершения генерации.
- [ ] Тесты подтверждают идентичность поведения с оригинальным CLI в части формирования запросов.
